3. Enterprise Storage Solutions

DAS (direct-attached storage)

就是本地直連硬盤。
● 好處：easy to configure，inexpensive
● 壞處：server 之間的硬盤互相孤立，而部署起來也欠缺靈活性，擴展亦有限。

NAS (network attached storage)

作為 file server.
● Windows : SMB / CIFS
● Linux : NFS
● 壞處：比 SAN 慢，對一些 data-intensive 的情況不太適用，比如 SQL server.

SAN （storage area network)

比 NAS 高級少少，可以用到 cluster 等功能。
● 好處：high level of redundancy
● 壞處：more expensive, requires specialized skills


需要用到一些特別的設備
● HBA (host bus adapter)：特別的介面卡 (或 switch)，可以做 redundancy.
● LUN (logical unit number)：SAN 用來儲存資料的邏輯盤叫做 LUN.


註：現在 NAS 與 SAN 都難以區分，就只有接入方式不同：
SAN 用 Fibre Channel over Ethernet, or iSCSI
NAS 用 SMB, or NFS

Scenario

● DAS：最簡單，少成本。
● NAS：互補了 DAS 和 SAN 的不足。
● SAN：最高效率，最多功能。

Block-level vs. File-level storage

● Block-level storage：傳輸快。
● File-level storage：傳輸慢。

Fibre channel

所需元件：
● SAN，
● HBA card (attached to computer)，
● Fibre Channel switch

iSCSI

又稱為 IP-SAN，所需元件：
● IP network，
● iSCSI targets (即係 server)，
● iSCSI initiators (即係 client)，
● iSCSI qualified name / IQN (即係 iSCSI device 的 ID)。

MPIO

很多網卡都內建的功能，Multi-Path IO，一張網卡有兩個 RJ45 (兩個 IP)，backup 連去 server 條 path。

Scale-Out File Server

多個 file server 同時連着一個大 volume，client 就連上隨便一個 file server 都可以存取到 volume 內的資料。

SMB

SMB 的 share profile :
● Quick：一般使用。
● Advanced：比 Quick 多了 data classification 和 quota，需要先安裝 FSRM. (file server resource manager)
● Application：適合用於 Hyper-V 和 database 等程式。

NFS

NFS 用 Kerberos V5 作為驗證機制。
NFS 的 share folders 叫 Exports。
