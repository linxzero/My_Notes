PKI 公鑰結構

PKI

(Public Key Infrastructure)
PKI is a set of roles, policies, and procedures needed to create, manage, distribute, use, store & revoke digital certificates and manage public-key encryption.

Digital Signature

Code Signing

在官網下載的 App，都會有 Code Signing 來保證他的 Authenticity 和 Integrity.
比如對在官網下載的 KMPlayer.exe 右鍵內容，就會見到他的 Digital Signatures，可以知道是由 Symantec 公司核查此軟件的真實性。

Digital Certificate

  • 用來核實信息來源的真確性。
  • 建立加密的通道，來傳遞保密資料。
  • Digital Certificate 包含用戶的基本信息以及他的 public key.

Algorithm

There are 3 DSS (Digital Signature Standard) algorithms that are used for generating and verifying digital signatures:

DSA

Signature generation is fast, but signature verification is slow.

RSA

Signature verification is fast, but signature generation is slow.

ECDSA

(Elliptical Curve Digital Signature Algorithm)
橢圓曲線配合 DSA，加快運算速度，而且生成更小的 signature.

Process of Digital Certificate

Bob ⇒

  1. Bob is going to send an order confirmation to Alice. His computer creates a Hash of the confirmation message.
  2. Then the Hash is encrypted with Bob's Private Key.
  3. This encrypted Hash, called Digital Signature, is appended to the confirmation message, and sent to Alice over internet.

Hash → Encrypt → Attach

⇒ Alice

  1. Alice accepts the confirmation msg with Signature, and obtains Bob's Public Key.
  2. Her computer decrypts the signature using the Public Key. Then a Hash value is revealed.
  3. Finally, she compares the decrypted Signature Hash to the hash of msg created by her computer. If they match, the message is authentic.

Public Key → Decrypt → Compare



Authority and Trust System

CA

(Certificate Authority)
第三方信任機構，他用 Private Key 來簽署 PKI cert，來證明個人或實體的真實身份。
PKI cert 內包含個人或實體的 Public Key、目的、CA、日期以及演算法。
