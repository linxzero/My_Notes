Cryptography 密碼學

保密溝通 (Securing Communication) 四要素

1. Confidentially

"Receiver"
只有認可的接收者能讀到訊息內容，就算中途被人截取也只是加密後的訊息，無法獲取訊息內容
加密技術可透過 Symmetric or Asymmetric encryption algorithms 來完成

2. Data Integrity

"Integrity"
訊息內容要完整無缺，任何的更改都能被發覺
透過 MD5 or SHA hash-generating algorithms 來完成
( MD5 - Message Digest version 5；SHA - Secure Hash Algorithm )

3. Origin Authenticity

"Sender"
保證訊息是由信中的發送人本人發出的，不是偽造的訊息
用 HMAC 來完成
( HMAC - Hash Message Authentication Code )

4. Non-Repudiation

"不能否認"
承上啟下，訊息含有發送人的獨有特徵，或者簽署，一定不是其他人發出的，所以無法抵賴

Integrity & Authenticity

Hash

  • One-way：不可逆的，不能由 hash 解成原文
  • Fixed-Length：任何長度的訊息都會消化成固定長度的 hash

MD5

(Message Digest version 5)
由 Ron Rivest 於 1992 年發布，生成 128-bit 長的 hash

SHA-1

(Secure Hash Algorithm 1)
由 US. NIST 於 1994 年發布，與 MD5 很相似，生成 160-bit 長的 hash

SHA-2

(224，256，384，512)
由 US. NIST 於 2001 年發布，生成 224-bit，256-bit，384-bit，512-bit 長的 hash

HMAC

(keyed-Hash Message Authentication Code)
光憑普通人都識的 Hash 技術，只能防止訊息在中途被意外更改，防不了 man-in-the-middle attacks。此時就需要 HMAC，他就是在訊息被消化時，加入只有 Sender 與 Receiver 知道的 Secret Key

Confidentiality

Key

Key Length
又稱為 Key Size，單位為 bit

Key Space
Key 的數量，例如 Key Length 為 10-bit，Key Space 為 1024 條

Symmetric encryption algorithm

對稱加密，用同一條 key 對資料加密和解密。
在建立溝通時，每個 communicating party 都要知道同一條 key，叫 Pre-shared Key (又叫 Secret Key)

DES

(Data Encryption Standard)
Timeline - 1976
Key size - 56 bits
Block size- 64 bits
Speed - Medium
Crack - 6.4 days

3DES

(Triple DES)
Timeline - 1977
Key size - 112 and 168 bits
Block size - 64 bits
Speed - Low
Crack - 4.6 billion years

AES

(Advanced Encryption Standard)
Timeline - 2001
Key size - 128, 192, 256 bits
Block size - 128, 192, 256 bits
Speed - High
Crack - 149 trillion years

SEAL

(Software-optimized Encryption Algorithm)
Timeline - 1997
Key size - 160 bits
It is a Stream Cipher
Speed - High
Crack - Unknown

RC

(Rivest Cipher)
RC2 - 1987 - 40 and 64 bits - Block cipher
RC4 - 1987 - 1 to 256 bits - Stream cipher
RC5 - 1994 - 0 to 2048 bits - Block cipher
RC6 - 1998 - 128, 192, 256 bits - Block cipher
( RC4 is widely used in SSL)

總結

  • 好處：佔較少資源，速度較快，通常用於加密大量資料，如 VPN traffic
  • Block cipher：每次對特定長度的 plaintext 加密，而通常密文為 64-bit 或 128-bit
  • Stream cipher：每次對 1 Byte 或 1 Bit 的 plaintext 加密

Asymmetric encryption algorithm

非對稱加密，用不同的 key 對資料加密和解密，分別叫 Public Key 和 Private Key。用公鑰加密後只能用私鑰解密，反之私鑰加密後只能用公鑰解密



加密 & 解密 (Cryptography & Cryptanalysis)

加密

Substitution cipher :
根據對應表把原文編譯成密文

Transposition cipher :
根據規則把原文重新編排次序，成為密文

Polyalphabetic cipher :
二維對應表的 Substitution cipher

解密

Brute-force method :
暴力破解，即是窮舉法

Ciphertext method :
不斷收集密文，再交電腦分析

Known-Plaintext method :
既收集密文，又知道訊息內一定含有某些資訊，比如日期、收件人等等

Meet-in-the-Middle method :
破解者就在明文與密文之間，收集無窮數據
