Cryptography 密碼學

保密溝通 (Securing Communication) 四要素

1. Confidentially

"Receiver"
只有認可的接收者能讀到訊息內容，就算中途被人截取也只是加密後的訊息，無法獲取訊息內容
加密技術可透過 Symmetric or Asymmetric encryption algorithms 來完成

2. Data Integrity

"Integrity"
訊息內容要完整無缺，任何的更改都能被發覺
透過 MD5 or SHA hash-generating algorithms 來完成
( MD5 - Message Digest version 5；SHA - Secure Hash Algorithm )

3. Origin Authenticity

"Sender"
保證訊息是由信中的發送人本人發出的，不是偽造的訊息
用 HMAC 來完成
( HMAC - Hash Message Authentication Code )

4. Non-Repudiation

"不能否認"
承上啟下，訊息含有發送人的獨有特徵，或者簽署，一定不是其他人發出的，所以無法抵賴

Integrity & Authenticity

Hash

  • One-way：不可逆的，不能由 hash 解成原文
  • Fixed-Length：任何長度的訊息都會消化成固定長度的 hash

MD5

(Message Digest version 5)
由 Ron Rivest 於 1992 年發布，生成 128-bit 長的 hash

SHA-1

(Secure Hash Algorithm 1)
由 US. NIST 於 1994 年發布，與 MD5 很相似，生成 160-bit 長的 hash

SHA-2

(224，256，384，512)
由 US. NIST 於 2001 年發布，生成 224-bit，256-bit，384-bit，512-bit 長的 hash

HMAC

(keyed-Hash Message Authentication Code)
光憑普通人都識的 Hash 技術，只能防止訊息在中途被意外更改，防不了 man-in-the-middle attacks。此時就需要 HMAC，他就是在訊息被消化時，加入只有 Sender 與 Receiver 知道的 Secret Key

加密 & 解密 (Cryptography & Cryptanalysis)

加密

Substitution cipher
根據對應表把原文編譯成密文

Transposition cipher
根據規則把原文重新編排次序，成為密文

Polyalphabetic cipher
二維對應表的 Substitution cipher

解密

Brute-force method
暴力破解，即是窮舉法

Ciphertext method
不斷收集密文，再交電腦分析

Known-Plaintext method
既收集密文，又知道訊息內一定含有某些資訊，比如日期、收件人等等

Meet-in-the-Middle method
破解者就在明文與密文之間，收集無窮數據
